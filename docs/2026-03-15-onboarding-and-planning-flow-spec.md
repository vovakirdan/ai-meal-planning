# Onboarding And Planning Flow Spec

**Date:** 2026-03-15  
**Status:** synced with current MVP implementation  
**Scope:** onboarding, weekly planning, recipes, shopping, review, reminders, settings

## Product Shape

- Бот работает как household meal planner в Telegram.
- В рамках одного household все едят общий набор блюд.
- Раздельная готовка не поддерживается.
- Household members нужны, чтобы учитывать пересечения вкусов, ограничений и персонального feedback.
- Reminder-настройки живут на уровне `telegram_user`.
- Pantry живет на уровне household.

## Current Public Commands

- `/start` — onboarding
- `/plan` — создать или пересоздать недельный план
- `/week` — открыть текущую неделю и редактировать блюда
- `/recipe` — открыть рецепт выбранного блюда
- `/ingredients` — открыть ингредиенты выбранного блюда
- `/shopping` — собрать корзину по активной неделе
- `/review` — оставить feedback по блюдам
- `/settings` — изменить постоянные настройки
- `/help` — показать справку
- `/cancel` — остановить активный текстовый сценарий

## Onboarding

### Entry Point

- `/start` запускает guided Q&A onboarding.
- Onboarding ощущается как короткий бытовой диалог, а не как длинная анкета.
- Если onboarding уже завершен, `/start` не запускает его заново и предлагает перейти к `/plan`.

### Household-Level Data

На уровне household бот собирает:

- количество человек;
- сколько приемов пищи в день обычно планировать;
- включать ли десерты;
- `repeatability_mode`.

### Meal Template Rules

- Пользователь выбирает количество приемов пищи в диапазоне `2-5`.
- По умолчанию используется `3`.
- Десерты задаются отдельным флагом.
- Внутри системы это разворачивается в фиксированные meal slots.

Внутреннее отображение:

- `2` = `breakfast`, `dinner`
- `3` = `breakfast`, `lunch`, `dinner`
- `4` = `breakfast`, `lunch`, `dinner`, `snack_1`
- `5` = `breakfast`, `lunch`, `dinner`, `snack_1`, `snack_2`
- `dessert` идет отдельным флагом поверх шаблона

### Household Members

После household-level блока бот отдельно проходит анкету по каждому участнику.

Для каждого участника собираются:

- имя;
- ограничения;
- любимые кухни;
- свободная заметка `что важно учитывать`.

### Reminders During Onboarding

Напоминания настраиваются сразу в onboarding:

- ежедневное напоминание на оценку блюд;
- еженедельное напоминание на составление следующего плана.

Обе настройки хранятся на уровне `telegram_user`.

### Pantry Step

- В конце onboarding бот предлагает опционально заполнить pantry.
- Pantry не является обязательной частью onboarding.
- Для MVP используются состояния `есть` и `мало`.
- Дополнительно можно указать примерный остаток или свободную пометку.

### Onboarding Exit

После завершения onboarding бот сразу предлагает перейти к `/plan`.

## Weekly Planning

### Default Logic

- План всегда строится на явный диапазон дат.
- Таймзона по умолчанию: `Europe/Moscow`.
- При запуске в середине недели бот по умолчанию предлагает следующую полную неделю.
- Полная неделя определяется как `понедельник-воскресенье`.

### Week Context Before Generation

Перед генерацией бот уточняет:

- диапазон дат;
- оставить ли текущий household meal template;
- настроение недели;
- weekly notes;
- учитывать ли pantry.

Из этого обязательны:

- диапазон дат;
- подтверждение meal template.

### Draft Generation

- После сбора weekly context бот создает `draft` недели в БД.
- Затем ИИ генерирует weekly menu skeleton.
- Draft сохраняется сразу, чтобы пользователь мог вернуться после ошибки или обрыва сети.

### `/week`

- `/week` открывает текущий активный план.
- Если есть `draft`, открывается он.
- Если `draft` нет, открывается последняя `confirmed` неделя.
- Обзор показывает неделю кратко по дням и слотам.

### Editing Flow

Редактирование происходит иерархически:

- день;
- прием пищи;
- блюдо.

На уровне дня доступны:

- просмотр приемов пищи;
- `Пересобрать день`.

На уровне приема пищи доступны:

- просмотр блюд;
- `Пересобрать прием пищи`.

На уровне блюда доступны:

- `Подобрать замену`;
- умные AI-подсказки для локальной корректировки;
- `Своя правка`;
- `Лайк`;
- `Не предлагать`.

### Local Edit Rules

- Локальная правка пересобирает только выбранное блюдо.
- Локальная правка не запускает автоматический пересчет баланса дня.
- Более широкая пересборка происходит только по явному действию пользователя.

### Plan Confirmation

- Draft подтверждается целиком кнопкой `Подтвердить неделю`.
- После подтверждения план остается редактируемым.
- После подтверждения бот в фоне прогревает recipe snapshots для блюд недели.

## Dish Modeling

### Canonical Dish Layer

- Каноническое блюдо хранится как сущность `dish`.
- Ингредиенты хранятся как сущности `ingredient`.
- При подтверждении плана блюда канонизируются и получают `dish_id`.

### Snapshot Layer

- В `planned_meal_item` хранится snapshot конкретного блюда в рамках текущей недели.
- Snapshot может отличаться от канонического `dish` за счет adaptation notes и локальных правок.
- При существенном изменении snapshot старая каноническая связь сбрасывается и создается заново.

### Dish Identity Rules

- Небольшие адаптации рецепта не создают новый `dish`.
- Изменение количества ингредиента, жирности, остроты или масла остается в рамках того же `dish_id`.
- Замена ключевой основы блюда, например `кальмар -> курица`, считается новым блюдом.

### Household Policies

- Household-level правила по блюдам хранятся отдельно от dish history.
- Сейчас используются как минимум два правила:
  - `favorite`
  - `never_again`
- Их можно снимать через `/settings`, не удаляя само блюдо и историю feedback.

## Recipes

### Viewing

- `/recipe` открывает дни активной недели.
- Внутри дня пользователь видит плоский список блюд с пометкой meal slot.
- Выбор блюда открывает полный рецепт.

- `/ingredients` использует тот же flow, но показывает только ингредиенты.

### Recipe Materialization

- Recipe details генерируются лениво и кэшируются в snapshot блюда.
- После подтверждения недели включается background warm-up, чтобы рецепты были готовы заранее.

### Recipe Feedback

- В карточке рецепта есть действие `Исправить рецепт`.
- Пользователь может описать проблему свободным текстом.
- Бот пересобирает рецепт для текущего snapshot блюда, не заменяя само блюдо.

## Shopping

### `/shopping`

- `/shopping` собирает корзину по активной неделе.
- Если есть `draft`, используется он.
- Если `draft` нет, используется последняя `confirmed` неделя.

### Basket Rules

- Корзина агрегирует одинаковые ингредиенты между блюдами.
- Если в нескольких блюдах есть один и тот же продукт, он суммируется.
- Если pantry содержит точный остаток в той же единице, он вычитается.
- Если pantry неточный, продукт может попасть в отдельную логику `проверить дома`.

- В корзине не показываются рецепты и не дублируются блюда.
- Это именно shopping basket, а не recipe dump.

## Review And Feedback

### `/review`

- `/review` работает по последней подтвержденной неделе.
- Пользователь сначала выбирает день.
- Затем feedback идет по схеме `блюдо -> участник household`.

### Verdict UX

Сейчас verdict делается тремя кнопками:

- `👎`
- `❤️`
- `👍`

Правила:

- `❤️` означает `favorite`;
- `👍` означает нормальное блюдо, которое можно повторять;
- `👎` открывает дополнительный шаг комментария или пропуска.

### Comments

- Комментарий по блюду опционален.
- Если комментарий есть, он нормализуется моделью и попадает в generation memory для будущих недель.

## Reminders

### Daily Reminder

- Scheduler проверяет локальную дату и время пользователя.
- Daily reminder отправляется только если на этот день есть блюда в подтвержденной неделе.
- Сообщение ведет в `/review`.

### Weekly Reminder

- Weekly reminder учитывает день недели и время.
- Если уже есть `draft`, бот предлагает продолжить через `/plan`.
- Если `draft` нет, предлагает составить новый план.

## Settings

### `/settings`

`/settings` управляет постоянными настройками без повторного onboarding.

Разделы:

- `Семья`
- `Участники`
- `Планирование`
- `Напоминания`
- `Запасы`
- `Правила по блюдам`

### Supported Edits

Сейчас через `/settings` можно:

- добавить участника;
- переименовать участника;
- отключить и снова включить участника;
- изменить ограничения, любимые кухни и заметку участника;
- изменить meal count, desserts flag и repeatability mode;
- изменить ежедневное и еженедельное напоминания;
- добавить, обновить и удалить pantry items;
- снять household-level rules `favorite` и `never_again`.

### Large Pantry Handling

- Раздел `Запасы` уже пагинируется.
- Это защищает UI от ошибок Telegram при большом количестве pantry items.
